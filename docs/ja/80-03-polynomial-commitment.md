# 多項式コミットメント — 封筒に入れて封をする

[← 算術回路](./80-02-arithmetic-circuits.md) | [次: Plonk →](./80-04-plonk.md)

## コミットメントとは

**コミットメント（Commitment）** は暗号学の基本構成要素で、2つの性質を持つ:

1. **束縛性（Binding）**: コミットした後に値を変更できない
2. **隠蔽性（Hiding）**: コミットメントから元の値を復元できない

日常の例え: 封筒に答えを書いた紙を入れて封をして渡す。後から封を開ければ答えを確認できるが、封をした後に中身を書き換えることはできない。

## 多項式コミットメントの役割

ZK証明では、witness（秘密の入力値と中間値）を**多項式**として表現し、その多項式にコミットする。

なぜ多項式か:
- 回路の全制約を1つの多項式の等式に集約できる（後述のPlonkで詳しく説明）
- 多項式の性質（次数、評価値）を使って効率的に検証できる
- Schwartz-Zippelの補題: 次数 d の異なる2つの多項式が同じ点で一致する確率は d/|F| （有限体のサイズが大きければ無視できるほど小さい）

## KZGコミットメント

MidnightのPlonkが使っている多項式コミットメント方式。Kate-Zaverucha-Goldberg (2010) の頭文字。

### 準備: 楕円曲線上の点

楕円曲線の生成元を G とする。スカラー s に対して sG は計算できるが、sG から s を逆算するのは（離散対数問題により）計算上不可能。

### Trusted Setup で作られるもの

秘密の値 τ（tau、toxic waste）を使って以下を計算し、公開する:

```
SRS = [G, τG, τ²G, τ³G, ..., τⁿG]
```

τ 自体は破棄される（MPCセレモニーで保証）。

### コミットの手順

多項式 f(x) = a₀ + a₁x + a₂x² + ... + aₙxⁿ に対して:

```
C = a₀·G + a₁·(τG) + a₂·(τ²G) + ... + aₙ·(τⁿG)
   = f(τ)·G
```

SRSの各項に係数を掛けて足し合わせるだけ。τ を知らなくても f(τ)·G は計算できる。

結果は楕円曲線上の**1つの点**。多項式の次数に関係なく固定サイズ（BLS12-381では48バイト）。

### 評価の証明（Opening Proof）

「f(a) = b である」ことを証明したい場合:

1. 商多項式を計算する: q(x) = (f(x) - b) / (x - a)
   - f(a) = b が本当なら、(x - a) は (f(x) - b) を割り切るので q(x) は多項式になる
   - f(a) ≠ b なら割り切れず、q(x) は多項式にならない → 証明を作れない
2. q(x) のコミットメント π = q(τ)·G を計算
3. 検証者は**ペアリング**を使って以下を確認:

```
e(C - b·G, G) = e(π, τG - a·G)
```

これは `f(τ) - b = q(τ) · (τ - a)` を楕円曲線上で検証している。

## ペアリング（Bilinear Map）

KZG検証の核心。2つの楕円曲線群 G₁, G₂ から目標群 GT への写像:

```
e: G₁ × G₂ → GT
```

重要な性質（双線型性）:

```
e(aP, bQ) = e(P, Q)^(ab)
```

これにより、楕円曲線上の点のまま「掛け算の等式」を検証できる。離散対数を解かずに多項式の関係を確認できるのがペアリングの威力。

BLS12-381は効率的なペアリング計算のために設計された曲線。

## バッチ検証

複数のコミットメントを同時に検証できる。Plonkでは1つの証明に複数の多項式コミットメントが含まれるが、ランダム線形結合でまとめて1回のペアリング計算で検証する。

これが「検証が O(1)」と言われる理由。正確には O(1) のペアリング計算 + 公開入力に比例する加算。

## 他のコミットメント方式との比較

| 方式 | 証明サイズ | 検証時間 | Trusted Setup | 量子耐性 |
|---|---|---|---|---|
| KZG | O(1) = 48バイト | O(1) ペアリング | 必要 | なし |
| FRI (STARKで使用) | O(log²n) | O(log²n) | 不要 | あり |
| IPA (Bulletproofsで使用) | O(log n) | O(n) | 不要 | なし |

KZGは証明サイズと検証時間が最小だが、Trusted Setupが必要という代償がある。

## まとめ

多項式コミットメントは「大量の値を1つの点に圧縮し、後から任意の位置の値を検証可能にする」仕組み。ZK証明システムの効率性の根幹を担っている。
